# 项目协作规范

## 对话前置要求

- 处理本项目的任何任务前，必须先阅读本文件和根目录下的 `README.md`。
- 后续工作只服务于第一组目标：数据预处理和经典机器学习算法分析。
- 除非用户明确要求，不实现第二组的深度学习部署，也不承担第三组的 PPT 制作和综合汇报工作。
- 可以为了第一组方案设计而简要提及深度学习，但不得偏离当前任务主线。

## 项目目标与分工

- 第一组：数据预处理和经典机器学习算法分析。
- 第二组：深度学习算法部署。
- 第三组：各算法对比总结分析、绘制适量图片及 PPT 制作。
- 当前用户负责第一组，因此代码、实验、文档和结果均应围绕第一组任务展开。

## 标准项目结构

```text
Data/
SourceFile/
Scr/
Results/
AGENTS.md
README.md
数据说明.txt
```

- `Data/`：存放解压后的 `.mat` EEG 数据，不在此目录保存代码或实验结果。
- `SourceFile/`：存放原始压缩包及其他未经处理的源文件。
- `Scr/`：存放 Python 源代码。项目沿用此目录名，不自行改为 `Src` 或 `src`。
- `Results/`：存放实验指标、模型输出、日志、表格和图片。

## 第一组技术范围

优先完成以下内容：

1. `.mat` 数据读取和数据结构检查。
2. 选取前 30 个 EEG 通道。
3. 去均值、带通滤波、陷波、时间窗截取和标准化等预处理。
4. 时域、频域、PSD、CSP 和 FBCSP 等经典特征提取。
5. LDA、SVM、逻辑回归、KNN、随机森林等经典分类器分析。
6. 分别开展 2 分类、4 分类和 6 分类实验。
7. 使用合理的交叉验证，避免数据泄漏。
8. 将指标、混淆矩阵及必要的结果图片保存到 `Results/`。

## 环境与依赖要求

- 新增依赖前，必须明确说明包名、用途、安装命令以及是否为必需依赖。
- 不得在未说明的情况下直接安装新的 Python 包。
- 优先使用项目已有依赖，避免为简单功能引入大型框架。
- 环境或依赖发生变化时，必须同步更新 `README.md`。
- 安装命令应优先使用：

```powershell
python -m pip install 包名
```

## 代码与文档要求

- 每次新建、删除或重命名代码文件，都必须同步更新 `README.md` 中的“代码文件说明”。
- 每个代码文件只承担清晰、相对独立的职责。
- Python 代码应包含必要的函数说明、输入维度和输出维度。
- 标签定义和类别筛选规则必须显式记录，不能依赖无法验证的默认顺序。
- 数据划分必须先于标准化、CSP、PCA 和特征选择等拟合过程，防止训练集信息泄漏到测试集。
- 不修改 `Data/` 中的原始数据文件；处理后的数据如需落盘，应保存到 `Results/` 或专门的结果子目录。

## 当前类别约定

依据 `数据说明.txt`，当前暂定第一个维度的类别顺序为：

```text
0：左手运动想象
1：右手运动想象
2：左腕运动想象
3：右腕运动想象
4：左臂运动想象
5：右臂运动想象
```

该顺序来自说明文字的排列，后续若获得更明确的数据字典，应及时核对并更新本文件和代码。
