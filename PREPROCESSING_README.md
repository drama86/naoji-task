# 预处理与特征提取小结报告

## 1. 任务目标

本阶段的目标是为运动想象（MI）EEG 经典机器学习流程确定一套尽可能稳健的前端方案，使 `2分类`、`4分类`、`6分类` 三种任务都能取得较好的综合效果。

本报告把前端两部分放在一起总结：

- 预处理
- 特征提取

重点回答三个问题：

1. 当前到底采用了什么方法；
2. 我尝试过哪些改进；
3. 最终哪套方案最适合作为主方案，为什么。

---

## 2. 数据与分析范围

当前数据来自 6 名被试，每个 `.mat` 文件中的 `eeg` 变量形状为：

```text
6 x 44 x 5000 x 20
```

含义为：

- 6 个类别
- 44 个通道
- 5000 个采样点
- 每类 20 个 trial

当前分析范围固定为：

- 只使用前 30 个 EEG 通道
- 采样率 `500 Hz`
- 单 trial 原始时长 `10 s`

当前主要任务为：

- `2class`：左手 vs 右手
- `4class`：左手、右手、左腕、右腕
- `6class`：全部六类运动想象

评价时主要参考指标为：

- `Balanced Accuracy`

---

## 3. 预处理方法总结

### 3.1 当前推荐预处理流程

当前综合实验后，推荐保留的预处理主线为：

```text
逐 trial 逐通道去均值
-> 50 Hz 陷波
-> 8-30 Hz 零相位带通滤波
-> 时间窗截取 1.5-5.5 s
-> 单 trial-单通道 z-score 标准化
-> 仅保留异常 trial 诊断，不自动剔除
```

换成配置语言就是：

```text
spatial_reference = None
notch_hz = 50.0
low_cut_hz = 8.0
high_cut_hz = 30.0
time_window_seconds = (1.5, 5.5)
normalize_mode = "zscore_per_trial_channel"
drop_flagged_trials = False
```

### 3.2 各步骤作用

#### 去均值

- 消除每个 trial、每个通道的直流偏置；
- 避免均值漂移直接污染时域统计特征。

#### 50 Hz 陷波

- 抑制工频干扰；
- 这是低风险、收益较稳定的常规清噪步骤。

#### 8-30 Hz 带通

- 聚焦运动想象最常见的 `mu/alpha` 与 `beta` 频段；
- 与后续 PSD 特征关注的频段一致。

#### 1.5-5.5 s 时间窗

- 去掉完整 10 秒 trial 中可能无关的前后段；
- 提高有效运动想象片段在输入中的占比；
- 是本阶段最有价值的预处理改进。

#### 单 trial-单通道 z-score

- 缩小不同 trial、不同通道的幅值差异；
- 让后续特征更关注波形结构和频带分布，而不是绝对幅值；
- 不会引入跨样本信息泄漏。

#### 异常值 trial 诊断

- 通过 `RMS` 和 `peak-to-peak` 对 trial 做稳健异常标记；
- 当前只输出 `trial_quality_summary.json`；
- 不默认自动删样本。

## 4. 预处理改进尝试与结果

### 4.1 时间窗对比

对比过的主要窗口：

- 完整 `0-10 s`
- `1.0-5.0 s`
- `1.5-5.5 s`

按每个任务的最佳 `Balanced Accuracy` 统计：

| 方案 | 2分类 | 4分类 | 6分类 |
|---|---:|---:|---:|
| 完整窗 `0-10 s` | 0.5500 | 0.3375 | 0.2653 |
| `1.0-5.0 s` | 0.6333 | 0.3333 | 0.2556 |
| `1.5-5.5 s` | 0.6292 | 0.3458 | 0.2639 |

结论：

- 截取中段时间窗明显优于完整 10 秒；
- `1.0-5.0 s` 在 `2class` 上略高；
- `1.5-5.5 s` 在 `4class` 上更好，同时 `2class` 几乎不吃亏；
- 从三种任务综合考虑，`1.5-5.5 s` 最平衡。

### 4.2 CAR 重参考

也尝试过 `CAR` 共均值重参考。

结论：

- 没有带来稳定提升；
- 在当前这份数据和当前特征体系下，`CAR` 不是统一最优选择；
- 因此当前推荐默认使用 `None`，即不启用 CAR。

### 4.3 异常值 trial 剔除

尝试规则：

- 用 `RMS` 与 `peak-to-peak` 做稳健异常标记；
- 合并规则为 `either`

剔除结果：

- 共剔除 `23 / 720` 个 trial

对比结果：

| 方案 | 2分类 | 4分类 | 6分类 |
|---|---:|---:|---:|
| `1.5-5.5 s` 不剔除 | 0.6292 | 0.3458 | 0.2639 |
| `1.5-5.5 s` 剔除异常 | 0.6250 | 0.3335 | 0.2692 |

结论：

- `6class` 略有提升；
- `2class`、`4class` 反而下降；
- 说明按幅值粗略删样本不够稳；
- 因此不建议作为默认流程。

### 4.4 预处理阶段最终结论

预处理这一步最终最值得保留的是：

- 合理带通与陷波
- 时间窗优化
- 单 trial 标准化

不建议默认保留的是：

- CAR
- 自动异常值剔除

## 5. 当前特征提取主线

### 5.1 基础特征

当前基础特征由两部分组成：

#### 时域统计特征

- mean
- standard deviation
- variance
- RMS
- peak-to-peak
- mean absolute value
- skewness
- kurtosis
- waveform length
- zero crossing rate

#### 频域特征

基于 Welch PSD 提取：

- alpha 频带绝对功率
- beta 频带绝对功率
- alpha 相对功率
- beta 相对功率
- 谱质心
- 归一化谱熵

这套基础特征的优点是：

- 稳定
- 可解释性强
- 对小样本更友好
- 不容易过拟合

## 6. 尝试过的特征提取改进

本阶段除基础特征外，还实现并测试了：

- `CSP`
- `FBCSP`
- `OVR-FBCSP + MIBIF`

并且都已经改造成：

- 在交叉验证训练折内部拟合
- 不会出现空间滤波或特征选择泄漏到测试折的问题

### 6.1 CSP

`CSP` 的核心思想是：

- 找到一组空间滤波器；
- 让某一类的投影方差尽量大；
- 同时让另一类的投影方差尽量小；
- 再用对数方差作为分类特征。

这里采用的是：

- 二分类直接做 CSP
- 多分类采用 `OVR`（one-vs-rest）扩展

### 6.2 FBCSP

`FBCSP` 是 CSP 的多带版本：

- 先把信号切分成多个子频带；
- 每个子频带单独做 CSP；
- 再把所有子频带上的 CSP 特征拼接起来。

这样做的优点是：

- 可以更细地捕捉被试和任务相关的最优频段；
- 对细粒度 MI 任务，理论上比单一宽带特征更有针对性。

### 6.3 OVR-FBCSP + MIBIF

参考了你提到的方案思路：

- 多带 `OVR-FBCSP`
- 后接 `MIBIF`（基于互信息的特征筛选）

`MIBIF` 的核心作用是：

- 对高维 FBCSP 特征按与标签的互信息大小排序；
- 只保留最有判别力的前 `k` 维特征；
- 以降低维度、减轻过拟合。

我们实际测试过：

- 不做特征选择
- `MIBIF 24` 维
- `MIBIF 36` 维

## 7. 特征提取实验结果总结

### 7.1 基础特征 vs CSP vs FBCSP

在当前推荐预处理下，用 `LDA` 做对比，结果保存在：

- [lda_feature_mode_compare.json](/home/epilogue/智能i信息/Results/02_feature_results/comparisons/feature_stage_compare/lda_feature_mode_compare.json)

结果如下：

| 特征模式 | 2分类 | 4分类 | 6分类 |
|---|---:|---:|---:|
| `basic` | 0.6250 | 0.3375 | 0.2639 |
| `csp` | 0.4833 | 0.3063 | 0.2333 |
| `fbcsp` | 0.5417 | 0.3604 | 0.2639 |

结论：

- `CSP` 整体不如基础特征；
- `FBCSP` 在 `4class` 上有优势；
- 但 `FBCSP` 在 `2class` 上明显变差；
- 所以它不是统一最优主线。

### 7.2 更接近参考方案的 OVR-FBCSP

进一步把 FBCSP 改成更接近参考方案的形式：

- 预处理频带放宽到 `4-40 Hz`
- 子频带采用 9 个 `4 Hz` 无重叠频带
- 每带做 `OVR-CSP`

代表性结果如下：

#### 纯 OVR-FBCSP + LDA

- `2class`：`0.5375`
- `4class`：`0.2979`
- `6class`：`0.2722`

说明：

- `6class` 比基础特征略高；
- 但 `2class`、`4class` 明显下降。

### 7.3 OVR-FBCSP + MIBIF

进一步测试了 `MIBIF`。

#### `MIBIF 24` 维 + LDA

在实际运行中，`2class` 和 `4class` 都没有拉起来：

- `2class`：`0.5458`
- `4class`：`0.2917`

#### `MIBIF 36` 维 + LDA

结果保存在：

- [fbcsp_mibif36_lda_only.json](/home/epilogue/智能i信息/Results/02_feature_results/comparisons/feature_stage_compare/fbcsp_mibif36_lda_only.json)

结果为：

| 方案 | 2分类 | 4分类 | 6分类 |
|---|---:|---:|---:|
| `OVR-FBCSP + MIBIF36 + LDA` | 0.5375 | 0.3000 | 0.2653 |

结论：

- `6class` 接近基础特征；
- `2class`、`4class` 仍然明显不如基础特征主线；
- 所以即使加入 `MIBIF`，也没有成为综合最优方案。

## 8. 为什么参考方案没有成为统一最优

这部分很关键。

从理论上说，`OVR-FBCSP + MIBIF` 是合理路线，尤其适合：

- 多分类
- 空间模式接近
- 频带差异微弱

但在当前这份数据上，它没有成为统一最优，主要原因有三点。

### 8.1 样本量偏小，高维空间滤波特征不稳定

当前每个被试每类只有 20 个 trial，总样本量有限。  
而 `OVR-FBCSP` 会带来很多子带、很多空间分量，再叠加 `OVR` 多分类展开，特征维度会快速变大。

结果就是：

- 理论判别力变强；
- 但估计方差也变大；
- 对 `2class`、`4class` 更容易不稳。

### 8.2 互信息筛选本身也依赖小样本估计

`MIBIF` 不是“必然提升”的，它也要在训练折内估计互信息。  
当样本不大、类别边界又不够清晰时：

- 互信息排序可能不稳定；
- 选出来的“最优特征”不一定对测试折同样有效。

### 8.3 当前目标不是只追某一个任务，而是三种任务都要兼顾

如果只看 `6class`，`OVR-FBCSP` 的确偶尔会略好；  
但你的目标是：

- `2class` 不能太差
- `4class` 也要尽量高
- `6class` 也要尽量好

在这个目标下，稳定性比“某一项偶尔略高”更重要。

## 9. 当前效果最好的方案是什么

如果按“统一主方案”来回答，当前最合适的是：

### 预处理

- 去均值
- 不用 CAR
- `50 Hz` 陷波
- `8-30 Hz` 带通
- `1.5-5.5 s` 时间窗
- 单 trial-单通道 z-score
- 不自动剔除异常 trial

### 特征

- 基础时域统计特征 + Welch PSD 频域特征

### 分类器

按任务分别选择最优分类器：

- `2class`：`Logistic Regression`
- `4class`：`Random Forest`
- `6class`：`LDA`

### 当前最好结果

| 任务 | 当前推荐主方案最佳分类器 | Balanced Accuracy |
|---|---|---:|
| `2class` | `basic + Logistic Regression` | 0.6292 |
| `4class` | `basic + Random Forest` | 0.3458 |
| `6class` | `basic + LDA` | 0.2639 |

---

## 10. 单任务局部最优与统一最优的区别

这里需要特别说明。

如果只看某一个任务，局部最优并不完全相同：

- `4class` 的单任务最高点曾出现在 `FBCSP + LDA`，约 `0.3604`
- `6class` 的单任务最高点曾出现在更接近参考方案的 `OVR-FBCSP + LDA`，约 `0.2722`

但这些方案的问题是：

- `2class` 会明显下降；
- `4class` 和 `6class` 的提升也不足以抵消整体不稳；
- 因此它们更适合作为“对照分支”，而不是统一主线。

所以：

- **单任务最优** 不等于 **综合最优**
- 当前应优先采用 **综合最优**

---

## 11. 最终结论

本阶段可以得出比较清晰的结论：

1. 预处理方面，最有价值的改进是时间窗优化与稳定标准化，而不是 CAR 或异常值剔除。
2. 特征方面，基础时域统计 + Welch PSD 仍然是当前最稳的主方案。
3. `CSP` 在这份数据上整体不如基础特征。
4. `FBCSP` 和 `OVR-FBCSP + MIBIF` 对 `6class` 或 `4class` 有一定潜力，但不能同时兼顾三种任务。
5. 如果目标是“让 2/4/6 分类都比较好”，当前最合理的统一方案仍然是：

```text
推荐预处理 + 基础时频特征 + 按任务选择最优浅层分类器
```

也就是说，当前最推荐的主线不是：

```text
OVR-FBCSP + MIBIF
```

而是：

```text
1.5-5.5 s 时间窗
+ 50 Hz 陷波
+ 8-30 Hz 带通
+ 单 trial z-score
+ 基础时域统计 + Welch PSD
+ 2class: Logistic Regression
+ 4class: Random Forest
+ 6class: LDA
```

这套方案是当前最稳、最容易解释、也最符合“三种任务都尽量兼顾”目标的方案。
